Probe the repository at the path below and write `ONBOARDING.md` at
that path.

**Arguments:** $@ (defaults to `.` if empty)

## Procedure

1. **Detect language and stack** by reading the presence of these files:
   - TypeScript/JavaScript: `package.json`, `tsconfig.json`,
     `pnpm-lock.yaml` / `yarn.lock` / `package-lock.json` / `bun.lockb`
   - Python: `pyproject.toml`, `requirements.txt`, `uv.lock`,
     `poetry.lock`, `Pipfile`
   - Go: `go.mod`, `go.sum`
   - Rust: `Cargo.toml`, `Cargo.lock`
   - Ruby: `Gemfile`
   - Java/Kotlin: `pom.xml`, `build.gradle*`

2. **Detect framework hints** by reading dependency lists:
   - Next.js, React, Vue, Svelte, Vite, Express, FastAPI, Flask,
     Django, Drizzle, Prisma, Vercel AI SDK, Tailwind, Playwright,
     Vitest, Jest, pytest, ruff, mypy.

3. **Read the README** (if present). Quote the one-paragraph
   description; do NOT paraphrase.

4. **Surface hot files** with `git log --pretty=format: --name-only
   --since=3.months | sort | uniq -c | sort -rn | head -15`. These
   are the highest-leverage files to read next.

5. **Check existing AI config** ‚Äî note presence of `AGENTS.md`,
   `CLAUDE.md`, `.cursor/`, `.claude/`, `.github/workflows/`.

6. **Check open issues** if it's a GitHub repo:
   `gh issue list --limit 10 --json number,title,labels`.

## Rules

- **Quote the README, don't paraphrase.** Saves accuracy questions
  later.
- **If the repo has its own `AGENTS.md` or `CLAUDE.md`, respect it.**
  Don't impose hack9 conventions on an unfamiliar repo. Note any
  tension in "Open questions".
- **Don't guess at the stack.** Write `unknown ‚Äî ask the team` if the
  signals are absent.
- **Cite every "key files" entry with `path:line`.** If you can't
  cite a line, you didn't really read it.
- **80-line cap on ONBOARDING.md.** It's a brief, not a tutorial.
- **No code edits from this prompt.** Onboarding is read-only
  investigation plus writing the brief.
